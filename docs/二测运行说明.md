# 二测运行说明

> 配套：`docs/数据采集规格说明.md`、`config/unesco_ich_keywords.v1.json`。长时间采集由 Hermes/Morn 跑，不交给 opencode session。

## 口径红线

- 原始数据不删除；清洗/打标只生成副本。
- `negative_terms` 只用于打标/排序，绝不用于采集时排除。
- `source_channel` / `source_type` 保留 search 与 hashtag 双通道，不混算。
- 缺失值留空，不补猜。
- 零结果、低结果照实记录在 report。

## 配置自检

```bash
cd /root/workspace/tiktok-heritage-crawler
python3 scripts/validate_keyword_config.py --config config/unesco_ich_keywords.v1.json
```

当前配置由既有首批数据反推，可能显示 `expected 45 projects, found 44` 的 warning。这个 warning 不能忽略；补齐官方工作表后再上最终大规模。

## 摸底采集口径

先不把正式上限定死为 1000。先做一次摸底采集，临时上限设 2000-3000，观察 44 项真实分布后再定正式 N：

- 若超过半数项目触顶：上调 N，或头部主要用可解析 hashtag 的平台 `videoCount/viewCount` 区分。
- 若绝大多数项目内容极少：正式分档重点放在低端，例如 0 / 几条 / 几十条 / 百条级。
- search 通道必须记录 `sample_rows_total`、`search_hit_cap_terms`、`search_has_more_at_stop`。触顶且 `has_more_at_stop=true` 表示平台仍有更多结果，只能解读为 `>=N`。
- report 中 `total_results` 是旧字段，仅代表采样行数，不是平台搜索总量；使用 `sample_rows_total` 和 search meta。

hashtag 的 `challengeInfo.stats.videoCount/viewCount` 只标注为“标签总规模（含噪声）”，不等于该非遗传播量。撞词严重项目（如剪纸、舞狮、皮影）必须结合 tag 下真实相关比例解释。

## region 与账号口径

建议摸底阶段采用单一海外基准 region，例如 US residential，目标是项目间可比。不要把多个 region 混成一个总体；如果后续要看“全球泛传播”，第二阶段再做 region 分层（如 US / EU / SEA），每个 region 独立跑同一词表。

每条视频和 report 都记录：

- `collector_account`
- `collector_account_role`：默认 `neutral`，避免长期中国非遗偏好号造成高估
- `proxy_region`
- `proxy_pool`
- `proxy_id`

采集时通过环境变量写入：

```bash
export COLLECTION_PHASE=pilot
export TIKTOK_REGION_POLICY=single-region-baseline
export TIKTOK_ACCOUNT_ID=neutral_us_01
export TIKTOK_ACCOUNT_ROLE=neutral
export TIKTOK_PROXY_REGION=US
export TIKTOK_PROXY_POOL=turnip-residential
export TIKTOK_PROXY_ID=us-res-001
```

摸底完成后做一次降质/地域偏置核对：挑热门、中等、冷门各若干项目，用干净号/IP 手动核对平台实际量级，与采集结果对比；确认没有风控降质或地域偏置后，量级分布才进入正式分析。

## 二测计划 dry-run

```bash

python3 run_second_test.py --dry-run --channels both --videos-per-term 30
```

只看前 1 项：

```bash
python3 run_second_test.py --dry-run --channels both --videos-per-term 2 --limit-projects 1
```

## 小 smoke test

会真实访问 TikTok，只跑前 1 项、每个 term 1-2 条：

```bash
cd /root/workspace/tiktok-heritage-crawler
source /root/workspace/tiktok-api-venv/bin/activate
set -a; source .env; set +a
python3 run_second_test.py --channels search --videos-per-term 1 --limit-projects 1
```

如果 search smoke 正常，再测 hashtag。hashtag endpoint 在 headless 下容易被 TikTok 返回空响应，优先用 `xvfb-run + TIKTOK_HEADLESS=false`：

```bash
TIKTOK_HEADLESS=false xvfb-run -a python3 run_second_test.py --channels hashtag --videos-per-term 1 --limit-projects 1
```

## 输出

`batch_collect.py` 会输出：

- `data/unesco_tiktok_<run_id>.ndjson`
- `data/unesco_tiktok_<run_id>.csv`
- `data/collection_report_<run_id>.json`
- `data/run_<run_id>_projects/*_{search,hashtag}.ndjson`

随后打标：

```bash
python3 quality_label.py data/unesco_tiktok_<run_id>.csv --out-prefix data/unesco_tiktok_<run_id>
```

会生成：

- `*_labeled.csv`
- `*_project_quality_summary.csv`
- `*_quality_report.json`
