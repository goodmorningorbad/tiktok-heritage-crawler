## 3. 已定决策（每条带「为什么」+「何时重新讨论」）

### 3.0 核心概念：存量 vs 触达（贯穿后续全部分析的原则，不可滑过）

手上的指标有**两类性质不同**的东西，之前易混用，现在明确分开。**「内容多」≠「传出去了」**——一个非遗可能有几千条视频，但大半 0 播放没人看，那是「内容多」但「传播失败」。

**病灶有两个同构的部位，两个都得写死，否则下「谁出海」结论时滑过任一个都会错：**

#### A. 数量轴：存量 ≠ 触达
| 类别 | 指标 | 衡量什么 |
|---|---|---|
| **存量类** | 视频条数、是否触顶、hashtag `videoCount` | **产出了多少内容**（有多少人发过） |
| **触达类** | playCount、digg、comment、share | **内容有没有被看到**（被多少人看到） |

真正的「传播成功」= 二者交叉。**存量 × 触达 二维框架：**

|  | 触达高（有人看） | 触达低（没人看） |
|---|---|---|
| **存量大**（内容多/触顶） | 真·规模化出海 | **大量产出但无人问津（自产自销/虚假繁荣）** |
| **存量小**（内容少） | 小而精破圈（单爆款） | 近乎隐形 |

> ⚠️ **「存量大 × 触达低」这一格是重点发现位**：可能是某些非遗靠中文内容自产自销、海外其实没人看——「我们以为传出去了，其实只是自己人发、自己人看」。

#### B. 语言轴：存量语言 ≠ 触达语言 ≠ 受众语言
| 层 | 看什么 | 含义 | 判出海中的角色 |
|---|---|---|---|
| **存量语言** | 中/英 term 采集量谁多 | 创作者**用什么语言发** | **只作背景，绝不单独判出海** |
| **触达语言** | 高播放视频是中/英内容 | **被看到的内容**是什么语言 | **辅证** |
| **受众语言** | 评论区谁在说话 | **看的人是谁** | **主判据** |

- **判断优先级：受众语言 > 触达语言 > 存量语言。**
- **反例（锚定，防误判）**：某非遗英文内容存量大（旧框架会判出海），但英文视频全 0 播放、有播放的是几条中文视频、评论区全中文 → **实际是内圈**。只看存量语言会误判成出海。

**为什么**：存量≠触达（数量）、存量语言≠触达语言≠受众语言（语言）是同一个病的两个部位。任一个被滑过，「内容多」就会被误读成「传出去了」，最终结论方向性错误。
**何时重新讨论**：不重新讨论，这是贯穿原则。


### 3.1 名录口径：锁定 44 项，2024-12 为准
- **决策**：本轮锁定 **44 项**，以 **2024 年 12 月名录为准**；2025 年变动（部分项目转列、咏春拳等新增）**不纳入本轮**（边界说明）。
- config 已打 `list_type`：

  | 类别 | 全称 | 数量 |
  |---|---|---|
  | `RL` | 人类非遗代表作名录 | 36 |
  | `USL` | 急需保护名录 | 7 |
  | `GSP` | 优秀保护实践名册 | 1 |

  - 赫哲族伊玛堪 = **USL**（2024-12 口径；2025 转列不计）。
  - **GSP（福建木偶戏传承计划）性质不同**：保护「传承机制/后继人才」而非遗产本身 → 分析时**单独识别、单列**，不与 RL/USL 混入规模比较。
- **为什么**：list_type 是核心论点的分层锚——RL 隐形与 USL 隐形性质完全不同（见 H2），混为一谈论点会被一击即破。
- **何时重新讨论**：若研究扩到 2025 名录，或决定纳入咏春拳等新增项目。

### 3.2 分层对象 = 非遗项目（44 个）
- **决策**：分层对象是**项目**，不是视频/term。
- **为什么**：研究问题是「哪些非遗出海/内圈」，项目是结论单元；视频是素材，term 是探针。
- **何时重新讨论**：若发现项目内部差异极大（同项目不同 term 走向相反），可能需降到 term 簇级别。

### 3.3 分层框架 = 存量 × 触达（二维，最终分层是此框架的结果，不是单看内容多少）

> 命名纠正：旧版「轴 A 触达轴 = 中英 term 采集量差异」是**错误命名**——采集量本质是**存量**（有多少人发），不是触达（有没有被看到）。见 §3.0。

#### 存量维度（衡量「产出了多少内容」）—— 数据已有，但只是框架的一半
- **search 采集量 / 是否触顶**：标 `ceiling_capped`/`exhausted`/`boundary`（见 §3.7、§4.1）。**只读作内容存量，不读作传播热度。**触顶 = 内容存量触及采集上限，**不等于高传播**。
- **hashtag `statsV2.videoCount`**（绝对存量规模）—— 脆弱，配三态标记，**缺失不硬填**：

  | 状态 | 含义 | 处理 |
  |---|---|---|
  | `clean` | 有干净唯一对应 hashtag | 可信，用 |
  | `noisy` | hashtag 撞词/含噪声（剪纸/舞狮/皮影 等通用词混入无关） | 标高估风险，谨慎用，报告注明 |
  | `unavailable` | 无可用 hashtag | 标「规模数据不可得」，不硬填 |

  - **绝不用 search collected_count 当规模**（被天花板压平，见 §4.1）；只用 hashtag videoCount。
  - `unavailable` 对冷门项目本身就是隐形佐证。

#### 触达维度（衡量「内容有没有被看到」）—— **独立必备维度，后续必须采集**
- 用每条视频的 **playCount / digg / comment / share** 刻画项目级触达：项目下视频是否有真实播放、头部视频触达量级、是否有破圈爆款 vs 全员低播放。
- **触达低 = 没传出去**，哪怕存量大、哪怕触顶。**这是之前框架缺的维，必须补。**

#### 语言构成（三层，判出海/内圈的核心，优先级见 §3.0-B）
| 层 | 数据来源 | 角色 |
|---|---|---|
| **存量语言** | 中/英 term 采集量谁多 | 背景，不单独判出海 |
| **触达语言** | 高播放视频是中/英内容 | 辅证 |
| **受众语言** | 评论区语言构成（见 §3.6） | **主判据** |

- **判出海优先级：受众语言 > 触达语言 > 存量语言。**单靠存量语言判出海会栽（§3.0-B 反例）。

- **为什么**：存量与触达数据质量、性质都不同；强行只用存量分档，会把「自产自销」误判成「出海」。
- **何时重新讨论**：若找到更可靠的存量源（官方 API/第三方榜单）或触达源，对应维度可升级。

### 3.4 分层产出形态
每项目一行（**存量 × 触达 二维，不是单看存量**）：
`项目 | list_type | 存量(触顶档 + hashtag规模+状态) | 触达(头部playCount/digg/comment量级) | 存量×触达象限 | 语言三层(存量语言/触达语言/受众语言) | 综合判断`
- **综合判断 = 存量×触达象限 + 语言三层（受众语言为主）共同决定**，绝不单看内容多少。
- 象限：真·规模化出海 / 自产自销(存量大触达低) / 小而精破圈 / 近乎隐形（见 §3.0-A）。
- **结论先按 list_type 分组再解读**：RL 隐形=功能化失败（重点），USL 隐形=常态，GSP 单列不比规模。

### 3.5 画像维度 = 5 个
- **2.1 内容**：表演/教学/商品/二创/搬运（分类体系后续细化）。
- **2.2 创作者**：本土vs海外 / 华人vs非华人 / 机构vs个人（可观测尽量做，不可观测诚实标注）。
- **2.3 语言**：标题/标签 中文/英文/混合。
- **2.4 传播形态**：原创vs搬运；是否破圈（非华人参与）。
- **2.5 受众** → 见 3.6。
- **为什么**：四个可观测维度全量可做，构成项目画像主体。
- **何时重新讨论**：内容分类体系跑通后细化标签；若某维度数据不可得，降级或剔除并记录。

### 3.6 受众维度定位 = 评论区语言构成，抽样，质性佐证
- **决策**：受众**不做精确比例**（TikTok 不给受众数据、评论区是极小比例互动者、语言分不清族裔）。降级为**评论区语言构成**（中文为主/英文为主/混合/几乎无评论），定位**有偏代理 + 小样本质性佐证**。
- **抽样规则**（框死范围，先评估成本再开采）：
  1. **不全量**。只挑分层结果里有代表性、能对比的**十几个项目**（规模大+英文触达的"疑似出海"、规模大+中文触达的"疑似内圈"、中等传播各 1–2 个）。
  2. 每项目只抽**头部几条高传播视频**，长尾空评论区不抽。
  3. 每条视频抓**评论区前 N 条**（够看语言构成倾向即可）。
  4. 产出 = 评论区语言构成，用来**印证** H2/H3 的出海/内圈判断。
- **执行顺序**：**必须排在分层之后**——先有分层才知道挑哪几个项目验证。
- **为什么**：精确受众做不出来又易被质疑；可观测+诚实标注的代理才站得住。
- **何时重新讨论**：若拿到更直接的受众信号（如平台开放数据），可升级。

### 3.7 数据洁净度：failed / zero_real / has_data / ceiling 严格分开（不可妥协）
- `failed`（限流/技术噪声，`empty_response`）→ 可重采，**永不计入低传播**。
- `zero_real`（真实零结果）→ 研究结论级的「真实零」。
- `has_data` → 有数据。
- `ceiling_capped`（触接口上限被压平，待 §6 待办落地）→ **不可当真实规模横向比较**。
- **为什么**：把限流噪声/真实零/触顶压平混进同一池，分层会建在脏数上，结论一推就倒。
- **何时重新讨论**：不重新讨论。这是红线。

## 4. 数据现状与硬约束

### 4.1 search 接口翻页硬顶 ≈ 第 7 页（offset 210 / ~180-200 条）—— 已实测坐实（2026-06-12）
- **证据链**：
  1. 197 term，collected_count max=196，无一超过 → 整齐天花板。
  2. 76 term（39%）`cursor_at_stop` 全部=210（整数台阶 30×7）。
  3. 逐页判决实验（`probe_ceiling.py`）：`chinesenewyear`（百万级）前6页满30+has_more=1，第7页突降8条+has_more=0 停188；`calligraphy` 同模式停192。两词独立复现。
  4. 排除限流：限流是 `empty_response`（整页空）；这里是 page7 *半页有数据*+has_more=0 → 接口逻辑硬顶，非 bot 检测。
- **平台行为**：第7页返「半页+has_more=0」**伪装自然采空**，`stop_reason=exhausted` 是被骗的。
- **能给/不能给**：
  - ❌ search collected_count **不能当规模**（高传播全压平）。
  - ✅ search 只能分两档：**远没到~180（真实低传播）** vs **顶到~180-200（高传播，具体多高未知）**。
  - 高传播之间的规模区分 → **必须靠 hashtag videoCount**。

### 4.2 各通道能给什么 / 不能给什么
| 通道 | 能给 | 不能给 |
|---|---|---|
| search `/api/search/item/full/` | 触达广度（中英可见度对比）、低/高传播两档、视频级 stats | 真实总量（>180 被压平）、规模横向比较 |
| hashtag `statsV2.videoCount` | 精确总量（字符串型），唯一可比规模源 | 非每词都有对应 hashtag；通用词撞词高估（noisy） |
| 视频级 stats（play/digg/comment/share） | **视频**热度 | **项目级**传播规模（视频热≠项目传播广） |
| 评论区 | 评论区语言构成（受众有偏代理） | 受众精确族裔/比例 |

### 4.3 baseline 数据现状（run_20260612_030425）

- ⚠️ collected_count 已被 §4.1 天花板压平，**不能直接当规模**；需派生 `ceiling_hit` 标记（见 §6）。

## 5. 已知局限专区（写报告时直接引用）

1. **search 接口上限**：>~180-200 的项目真实量被压平，规模不可横向比较；只能分「低传播 vs 高传播」两档。高传播内部区分依赖 hashtag。
2. **hashtag videoCount 含噪声**：通用词 hashtag（剪纸/舞狮/皮影等）撞词高估；部分项目无可用 hashtag → 规模「不可得」。每项目规模状态已标 clean/noisy/unavailable。
3. **受众用评论区代理**：评论区语言是**有偏的受众代理**，小样本、非统计比例，不代表全部受众。
4. **US 单 region**：仅 US residential 出口，结论限于「美区」，不可外推全球。
5. **中文词美区触达浅**：美区中文 term 普遍采到的量少于英文 term——这是发现也是局限（中文内容在美区算法下本就少曝光）。
6. **GSP 性质特殊**：福建木偶戏是保护计划而非遗产本身，不参与规模/传播横向比较。
7. **名录口径边界**：以 2024-12 名录为准，2025 变动未纳入。

### 接口翻页上限 & 边界态（已坐实，2026-06-12 补）

**已定事实**：
- search 接口存在翻页硬顶，约在第 6-7 页（offset 180-210）。到顶后返回半页 + has_more=0，**伪装成自然采空**。已用 chinesenewyear / calligraphy 等逐页实测坐实。
- 触顶判据（最终版，cursor 数值已降级为参考字段 `cursor_at_stop`，不参与判定）：
  - `hit_ceiling = 末页未满(<30) AND 倒二页满(>=28)+has_more=1 AND 停在采空/空页 AND 已进上限区(pages_fetched>=6 OR cursor>=180)`
  - 上限区排除项是关键：第 6 页之前 has_more=0 必是真采空，不算触顶。
- baseline 197 词：76 个 cursor=210 铁证触顶；39 个 cursor=180 存疑经逐页验证 = 38 触顶 + 1 采空。

**边界态发现（重要，影响判定稳定性）**：
- **藏医**是骑墙个例：真实结果量 ≈150，正好卡在接口第 5-6 页边界。
  - 验证脚本(acc4)：第 5 页 25 条 has_more=0 → 判 exhausted
  - 实采(acc3)：第 5 页满 30 条 has_more=1，第 6 页 3 条 → 判 ceiling_capped
  - 两次判定**都对**，差异源于词本身数据量在临界点上的采集抖动。
- **结论**：cursor≈150-180、结果量≈150 的词，**单次采集判定不稳定**，应标记为"边界态(boundary)"而非硬归 ceiling/exhausted 一类。涉及触达档归类时，对这类词需多次采集取众数或保守归为"≥150 触顶嫌疑"。
