# UNESCO 中国非遗在 TikTok 美区的传播研究
## 阶段进展与方法论汇报

> 汇报定位：**研究进展 + 方法论**，非成果展示。本阶段重点在于建立可靠的研究设计、坐实数据采集中的关键技术约束、并以严格的口径与质控保证后续分层分析的数据洁净度。
> 数据基准：baseline 采集 run（2026-06-12），辅以两轮逐页验证实验。

---

## 一、研究设计

### 1.1 研究问题
判断每一个 UNESCO 认定的中国非物质文化遗产项目，在 TikTok 美区是处于**「真出海」**（破圈到非华人 / 英文世界）还是**「内圈循环」**（华人 / 中文小圈层内传播），并为每个项目建立**传播画像**。

### 1.2 研究对象与口径
- **对象 = 非遗项目（44 个）**。视频与搜索词只是采集**素材与探针**，不是研究单元——结论落在项目层级。
- **名录口径锁定 2024 年 12 月名录**，2025 年的转列 / 新增（如咏春拳）不纳入本轮，作为明确的边界声明。
- 44 个项目按 UNESCO 名录类别分层：

  | 类别 | 全称 | 数量 |
  |---|---|---|
  | RL | 人类非遗代表作名录 | 36 |
  | USL | 急需保护名录 | 7 |
  | GSP | 优秀保护实践名册 | 1 |

  - **类别分层对核心论点至关重要**：RL 项目「隐形」意味着**功能化失败**（理应被代表、却没传播开）；USL 项目「隐形」是**常态**（本就濒危）。二者性质完全不同，绝不可混为一谈。
  - GSP（福建木偶戏传承计划）保护的是「传承机制」而非遗产本身，**单列识别，不参与规模横向比较**。

### 1.3 分层框架：规模 × 触达 双轴（数据基础不对等，区别处理）
- **轴 A — 触达轴（主轴，数据扎实）**：以同一项目下中文词 vs 英文词的采集量差异衡量「触达广度」。英文 >> 中文 → 疑似出海；中文 >> 英文 → 疑似内圈；双浅 → 低传播。
  - 关键优势：触达轴衡量的是**相对可见度**而非绝对总量，**不受后文所述的接口翻页上限影响**。
- **轴 B — 规模轴（辅轴，数据脆弱）**：以 hashtag 的 `videoCount` 衡量绝对规模，配 `clean / noisy / unavailable` 三态标记，缺失不硬填。

### 1.4 画像维度（5 个）
内容形态 / 创作者构成 / 语言 / 传播形态（四个可观测维度全量做）+ 受众（降级为评论区语言构成，抽样质性佐证，排在分层之后执行）。

> **方法论原则**：受众的精确族裔比例平台不提供、也无法可靠推断，故诚实降级为「有偏代理」，不做无法支撑的精确声明。

## 二、采集架构与规模

### 2.1 采集架构
- **采集通道**：TikTok search 接口（`/api/search/item/full/`），逐页翻页采集（每页 30 条）。
- **账号 / 代理池**：4 个独立账号，各自绑定美国 residential SOCKS5 代理出口（FL / CA / NM 多地），通过本地代理端口接入。
- **调度器（方案 A）**：账号轮换 + 冷却队列设计——
  - 最细粒度调度单元 =（项目，搜索词）；
  - 单账号连续采集 N 个词后主动冷却 90s 并轮换下一账号（预防性轮换，降低风控触发）；
  - 失败词自动重入队列、可重采；
  - `(项目, term)` 粒度落盘，保证可追溯。
- **运行环境**：headless 浏览器（xvfb 虚拟显示），独立 Python venv。

### 2.2 baseline 采集规模（run 2026-06-12）
| 指标 | 数值 |
|---|---|
| 搜索词（term）总数 | 197 |
| `has_data`（有数据） | 196 |
| `zero_real`（真实零结果） | 1 |
| `failed`（限流 / 技术失败） | **0** |
| 重采次数 | **0** |
| 采集视频（原始） | 32,579 条 |
| 采集视频（去重唯一） | 29,407 条 |
| 单词采集量 | 均值 166 / 中位 174 / 最大 196 / 最小 60 |

> **零失败、零重采**是本轮采集质量的核心指标：197 个词全部解析成功，没有任何限流或技术噪声混入数据池——这是后续判定能够干净进行的前提。

## 三、方法论发现（本阶段核心）

### 3.1 发现：search 接口存在隐蔽的翻页硬顶

**现象**：197 个词的最大采集量整齐地停在 196，无一突破。这种「整齐的天花板」不像自然现象，触发了对接口本身的怀疑。

**逐页验证实验**：设计单词逐页探针（`probe_ceiling.py`），对高传播词逐页记录每页返回的 `items / cursor / has_more`：
- `chinesenewyear`（百万级话题）：前 6 页每页满 30 条、`has_more=1`，**第 7 页突降至 8 条、`has_more=0`** 停在 188 条；
- `calligraphy` 同模式停在 192 条；
- 两词独立复现。

**结论（已坐实，非推测）**：search 接口在约**第 6–7 页（offset 180–210）存在翻页硬顶**。到顶后接口返回「半页 + `has_more=0`」，**伪装成自然采空**（`stop_reason=exhausted` 是被欺骗的结果）。

**为何不是限流？** 限流的特征是整页空响应（`empty_response`）；而这里是「半页**有数据** + `has_more=0`」，是接口逻辑层面的硬顶，不是风控拦截。两者在数据上可清晰区分。

### 3.2 处理：判据从「数值」升级为「行为模式」

接口硬顶意味着：**search 采集量不能直接当作传播规模**——所有高传播项目的真实量都被压平在 ~180–200。

最初的判定用 cursor 数值阈值（`cursor >= 180` 即判触顶），但逐页验证暴露了它的脆弱：同一数值既可能是触顶、也可能是真实采空。因此判据升级为**「has_more 突变模式 + 上限区排除」**：

```
触顶 (hit_ceiling) = 末页未满(<30 条)
                  AND 倒数第二页满(>=28 条) 且 has_more=1
                  AND 停止原因为采空 / 空页
                  AND 已进入上限区（已翻 >=6 页 或 cursor>=180）
```

- **「上限区排除」是关键**：第 6 页之前就 `has_more=0`，必然是真实采空，不算触顶。
- **cursor 数值降级为参考字段**（`cursor_at_stop`），不再参与判定。

### 3.3 进一步发现：「边界态」——判定的稳定性问题

在验证中发现一个更深的现象：**「藏医」一词，两次采集结果不同**——
- 一次第 5 页（cursor=150）即 `has_more=0`，判为「真实采空」；
- 另一次撑到第 6 页（cursor=180），判为「触顶」。

**两次判定都正确**。差异源于该词的真实结果量恰好 ≈150，**正卡在接口第 5–6 页的边界上**，采集时的微小抖动就会改变它落在哪一档。

**方法论意义**：结果量在临界点（cursor ≈ 150–180）的词，**单次采集判定本质不稳定**。强行将其钉死为「采空」或「触顶」，等于**用假确定性掩盖真实的不确定性**——后续分层会因此把一个可能高传播触顶的项目误判为「中等真实量」，造成方向性误判。

**处理决策**：为这类词设立**第三态「边界态（boundary）」**，明确标记为「待用其他方法定位」（结合 hashtag videoCount 或多次采集取稳定倾向），**而非硬归入任一确定档**。这保证了另外两档（确定触顶 / 确定采空）的洁净。

## 四、口径与质控

### 4.1 洁净度红线：四类严格分开，绝不混淆
| 类别 | 含义 | 处理原则 |
|---|---|---|
| `failed` | 限流 / 技术噪声（空响应） | 可重采，**永不计入低传播** |
| `zero_real` | 真实零结果 | 研究结论级的「真实零」 |
| `has_data` | 有数据 | 正常使用 |
| `ceiling_capped` | 触接口上限被压平 | **不可当真实规模横向比较** |
| `boundary` | 临界抖动，单次判定不稳 | 待其他方法定位，不硬归类 |

> **这是不可妥协的红线**：把限流噪声、真实零、触顶压平混入同一池，分层就建在脏数据上，结论一推就倒。

### 4.2 各采集通道「能给 / 不能给」边界
- **search 接口**：能给「触达广度（中英可见度对比）」「低 / 高传播两档」「视频级数据」；**不能给**「真实总量（>180 被压平）」「规模横向比较」。
- **hashtag `videoCount`**：能给「精确总量（唯一可比规模源）」；**不能给**「每词都有对应 hashtag」（通用词撞词高估，标 noisy；无 hashtag 标 unavailable）。
- **视频级 stats**：能给「单视频热度」；**不能给**「项目级传播规模」（视频热 ≠ 项目传播广）。
- **评论区**：能给「评论区语言构成（受众有偏代理）」；**不能给**「受众精确族裔 / 比例」。

### 4.3 研究文档纪律：决策与假设分区
研究纲领作为**活文件**维护，强制区分：
- **「已定决策」**（每条带「为什么」+「何时重新讨论」）；
- **「待验证假设」**（明确标注，未经数据验证不得当结论引用）；
- 任何新发现 → 更新对应区 + 追加修订记录。

> 例：「相当一部分 RL 项目在美区隐形 = 功能化失败」目前是**假设（H2）**，不是结论，待分层数据验证。

### 4.4 已知局限（提前声明，不掩盖）
1. search 接口上限 → 高传播项目真实量被压平，规模只能分「低 / 高」两档，高传播内部区分依赖 hashtag。
2. hashtag videoCount 含撞词噪声，部分项目规模「不可得」。
3. 受众用评论区语言代理，小样本、有偏、非统计比例。
4. 仅 US 单 region residential 出口，结论限于美区，不外推全球。
5. 名录以 2024-12 为准，2025 变动未纳入。
